feat(p2): API key auth + workspace registry + configurable embedding

P2 batch 1 — backend-only pieces from architecture-v2.md.

1) Optional API-key authentication (P1 leftover, backend-ready now)
   * core/auth_middleware.py: Starlette middleware that gates every
     request behind `X-API-Key` or `Authorization: Bearer <key>`. Public
     paths (/health, /docs, /redoc, /openapi.json) and CORS preflights
     bypass the gate. No-op when the setting is empty.
   * settings: new `OMNICODE_API_KEY` field (env-driven, default "").
   * main: install middleware after CORS so it can also serve the
     preflights it lets through.
   * 9 unit tests covering all branches.

2) Workspace registry (P2 step 4)
   * omnicode_core/workspace: thread-safe JSON-backed bookmark store
     persisted at `~/.kiro/codebase-mcp/workspaces.json`. Atomic writes
     via tmp-file rename. Auto-promotes a survivor when the active
     workspace is removed so callers always have a target.
   * api/v1/routers/workspaces.py: REST CRUD + activate + rename.
     `PUT /workspaces/{id}/activate` also flips the in-process
     `WORKING_DIR` and signals `requires_restart=true` so heavy
     services (FAISS, etc.) get re-initialised on the next process.
   * 8 unit tests covering registry round-trips and persistence.

3) Configurable embedding model (P2 step 7)
   * omnicode/search/engine.py now reads `EMBEDDING_MODEL` from settings
     instead of hardcoding `all-MiniLM-L6-v2`. Existing default is
     unchanged so behaviour stays identical out of the box.

4) Lint clean-up across all v2 packages
   * Fixed 14 pre-existing ruff errors in omnicode_core (unused
     imports, `from err` chaining, B007 loop var renames, ambiguous
     `l` -> `line`).
   * Extended CI lint scope to include `omnicode_core` and
     `omnicode_adapters` so future regressions are caught.
